1. 动态命令

使用此命令实用程序，您可以将命令声明为“动态”，这意味着在进入预览模式时将解析此命令。

要声明动态命令，请在命令开始标记后添加加号：+``<%+

就是这样，您的命令现在将仅在预览模式下执行。

这对于内部函数很有用，例如：tp.file.last_modified_date

Last modified date: <%+ tp.file.last_modified_date() %>

刷新问题

预览模式的一个“缺点”是它将渲染的笔记放在缓存中，以加快速度。

这意味着您的动态命令只会在您打开备忘录时呈现一次，但之后不会刷新。

如果要刷新它，则必须关闭注释以清除缓存并再次打开它。

3.空白控件

默认情况下，模板程序中的命令不会删除任何换行符。命令被替换为它们的值，仅此而已。

有时，在插入命令后进行一些空格控制会很有用，这正是此命令实用程序提供的。

让我们举个例子。以下模板：

<%* if (tp.file.title == "MyFile" ) { %> This is my file! <%* } else { %> This isn't my file! <%* } %> Some content ...

如果条件为 false，将生成以下输出（当条件为 true 时也会发生同样的情况），请注意空行：

This isn't my file! Some content ...

您可能希望删除执行命令生成的不产生任何输出的空行。
